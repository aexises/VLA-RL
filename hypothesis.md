# Краткое резюме

В данном отчёте предлагается **интеграция плотных вознаграждений** в алгоритм **Group Relative Policy Optimization (GRPO)** для обучения политики на визуально-языковых задачах манипуляции. Базовая постановка использует бинарное терминальное вознаграждение, тогда как предлагаемый подход сравнивает три формулировки:

1. `R_sparse` — бинарное терминальное вознаграждение
2. `R_dense` — плотное вознаграждение в стиле ReinboT
3. `R_clipped` — обрезанное плотное вознаграждение с порогом `τ_clip`

Основная идея состоит в том, что шаговый сигнал `r_t` позволяет лучше распределять credit assignment по траектории, а групповая нормализация в GRPO снижает дисперсию оценок преимущества.

## Формулировка тестируемой гипотезы

- **Гипотеза 1:** Плотное вознаграждение в GRPO улучшает эффективность обучения по сравнению с разреженным бинарным вознаграждением.
- **Гипотеза 2:** Различные компоненты плотного вознаграждения по-разному влияют на качество траекторий и скорость обучения.
- **Гипотеза 3:** Групповая нормализация снижает дисперсию градиентов и делает обучение стабильнее.

Каждая гипотеза должна проверяться в контролируемых экспериментах при одинаковых условиях обучения.

## Математическое обоснование

### Определение плотного вознаграждения

Плотное вознаграждение задаётся как шаговый сигнал

$$r_t = w_1 r^{\rm sub}_t + w_2 r^{\rm prog}_t + w_3 r^{\rm smooth}_t + w_4 r^{\rm final}_T$$

где:

- $r^{\rm sub}_t$ — награда за достижение подцелей
- $r^{\rm prog}_t$ — награда за прогресс по задаче
- $r^{\rm smooth}_t$ — штраф за резкие изменения действий
- $r^{\rm final}_T$ — терминальный индикатор успеха

Для clipped dense используется

$$r^{\rm clipped}_t = \max(r_t, \tau_{\rm clip}), \qquad \tau_{\rm clip} \in \{0, 0.1, 0.2, 0.5\}$$

### Кумулятивные возвраты

Кумулятивное вознаграждение для шага $t$:

$$R_{i,t}=\sum_{t'=t}^{T_i}\gamma^{t'-t}r_{i,t'}$$

### Групповая нормализация

$$\hat R_{i,t}=\frac{R_{i,t}-\mu}{\sigma}, \qquad \mu=\text{mean}(\{R_{j,u}\}), \ \sigma=\text{std}(\{R_{j,u}\})$$

где нормализация проводится по всей группе траекторий.

### Преимущества

Базовый вариант:

$$A_{i,t}=\sum_{t'=t}^{T_i}\hat R_{i,t'}$$

Опциональный вариант с дисконтированием:

$$A_{i,t}=\sum_{t'=t}^{T_i}\gamma^{t'-t}\hat R_{i,t'}$$

### Целевая функция GRPO

Оптимизация выполняется по стандартной objective-функции GRPO:

$$J(\theta)=\mathbb{E}\Bigl[\frac{1}{G}\sum_{i=1}^G\frac{1}{|\tau_i|}\sum_{t=1}^{|\tau_i|}\Bigl(\frac{\pi_\theta(a_{i,t}|s_{i,t})}{\pi_{\theta_{\rm old}}(a_{i,t}|s_{i,t})}A_{i,t}-\beta\Bigl(\frac{\pi_{\rm ref}(a_{i,t}|s_{i,t})}{\pi_{\theta_{\rm old}}(a_{i,t}|s_{i,t})}-1\Bigr)\Bigr)\Bigr]$$

## Алгоритм GRPO с плотным вознаграждением

```text
Вход:
- политика π_θ
- среда Env
- reward function r(s, a, s')
- параметры: G, ε, β, η, γ, число итераций N

1. Сэмплировать группу из G траекторий
2. Для каждого шага вычислить R_{i,t}
3. Выполнить групповую нормализацию R̂_{i,t}
4. Вычислить преимущества A_{i,t}
5. Обновить политику по objective GRPO / PPO-style clipped surrogate
6. Повторять до завершения обучения
```

## Предложенные эксперименты

Для проверки гипотез предлагается проводить контролируемые эксперименты на VLA-задачах манипуляции:

- **Окружения:** LIBERO, RoboTwin, CALVIN, а также другие релевантные робототехнические симуляции
- **Базовые методы:**
  1. `GRPO (sparse)`
  2. `GRPO (dense)`
  3. `GRPO (clipped dense)`
  4. `SFT only`

- **Метрики качества:**
  - Success Rate
  - Средняя кумулятивная награда
  - Скорость сходимости / sample efficiency
  - Стабильность поведения
  - Статистическая значимость

- **Анализ и аблации:**
  - sweep по `τ_clip`
  - аблации весов компонентов награды
  - сравнение с и без групповой нормализации
  - вариации `G`, `β`, `γ`, `clip_range`
  - анализ sample complexity

## Сравнение вариантов плотного вознаграждения

| Компонент | Описание | Ожидаемый эффект | Риски |
|---|---|---|---|
| Distance-to-goal | Награда за приближение к цели | Ускоряет движение к цели | Может искажать цель |
| Sub-goals | Награда за промежуточные этапы | Улучшает декомпозицию задачи | Требует ручного дизайна |
| Progress | Награда за прогресс по этапам | Улучшает долгосрочное credit assignment | Чувствительна к весам |
| Smoothness | Штраф за резкие действия | Делает траектории плавнее | Может замедлить достижение цели |
| Potential component | Потенциально-инвариантная форма | Теоретически безопаснее | Трудно подобрать потенциал |

## Заключение

Предложенная модификация GRPO с плотным и обрезанным плотным вознаграждением формирует проверяемую гипотезу: шаговый reward-сигнал должен ускорять обучение, улучшать credit assignment и снижать дисперсию оценок преимущества по сравнению с бинарным терминальным reward. Итоговая ценность подхода должна подтверждаться только контролируемыми экспериментами, статистической проверкой и анализом аблаций.
